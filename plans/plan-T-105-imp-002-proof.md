## Overview
The self-hosted proof artifact. Two-part work: (1) pick a trivial-but-real scope for IMP-002 and author its brief; (2) run the lifecycle agent against IMP-002 on a dedicated branch with the operator (you) doing the implementation step. ## Steps

### 0. Investigation deliverable (pre-run)
Write `plans/plan-T-105-imp-002-scope-investigation.md` documenting:
- **Chosen scope** for IMP-002. Candidate: "add `uv run orchestrator agents show lifecycle-agent@0.1.0` smoke to README First Run." Diff: ≤ 5 lines of README.md. Single file.
- **Expected task breakdown**: 1 task (T-001 in IMP-002's task list), simple file edit.

Get approval on the investigation plan before kicking off the real run.

### 1. Create `docs/work-items/IMP-002-lifecycle-proof.md`
Follow `.ai-framework/templates/improvement-proposal.md`. Fields:
- Identity table: IMP-002, Name: "Lifecycle self-hosted delivery proof", Status: `In Progress`, Priority: Medium, Date: 2026-04-18.
- Problem: "First concrete demonstration of AD-6 (eat our own dog food) — prove the lifecycle agent can drive a real repo change end-to-end."
- Scope: exactly one file change (README.md, specific line addition).
- Acceptance: "After orchestrator run, the README contains the new command line, tests pass, and the brief's Status is Completed."
- Keep it short; the point is the orchestration, not the change.

### 2. Create the dedicated branch
```bash
git checkout -b feat/imp-002-lifecycle-proof
git add docs/work-items/IMP-002-lifecycle-proof.md
git commit -m "docs: add IMP-002 work item for lifecycle proof"
```

### 3. Execute the orchestrator run
```bash
uv run orchestrator run lifecycle-agent@0.1.0 \
    --intake workItemPath=docs/work-items/IMP-002-lifecycle-proof.md \
    --follow
```

The agent proceeds through intake → task-gen → assignment → plan. At `implementation`, it pauses. Open a second terminal:
- Read `plans/plan-T-001-*.md` generated by the agent.
- Run `claude` (Claude Code), point it at the plan, land the exact edit.
- `git add README.md && git commit -m "feat: add agents-show smoke to README first run"`
- Signal back: `uv run orchestrator tasks mark-implemented T-001 --run-id <run-id> --commit-sha $(git rev-parse HEAD)`

The agent proceeds to `review`, reads the diff, emits `verdict=pass`, then `closure` flips IMP-002's Status to `Completed`.

### 4. Commit evidence
```bash
cp .trace/<run-id>.jsonl docs/work-items/evidence/IMP-002-trace.jsonl
# Scan for any leaked secrets in the trace before committing.
git add docs/work-items/evidence/
git commit -m "docs: add IMP-002 evidence trace"
```

Verify the committed trace does NOT contain:

## Edge Cases & Risks
- **Run crashes mid-way**: per v1 contract, partial runs aren't resumable. If the run crashes at e.g. `plan_creation`, delete the generated `tasks/IMP-002-*.md` + any `plans/plan-T-001-*.md` and re-run. Rare if T-101-104 are green.
- **Agent hallucinates off-scope changes**: review catches it (hopefully); if not, correction-bound trips. Worst case: redo with tighter prompt. Document any issues in the investigation plan's retrospective.
- **Trace contains sensitive content**: grep for `ANTHROPIC_API_KEY`, `Bearer`, email patterns before committing. If anything slips, redact and re-commit.
- **Commit SHA on signal**: the `--commit-sha` passed with the signal should match the commit the operator just created. If misaligned, the review reads the wrong diff — surface as review fail, bound trips eventually.
- **Run duration**: expect 2–5 minutes of real-world wall-clock, dominated by Anthropic latency and the operator's implementation step.
- **Cost**: ~$0.20 for the Anthropic side (matches T-104's estimate).
