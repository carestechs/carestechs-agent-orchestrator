## Implementation Steps

### Step 1: Create `pyproject.toml`
**File:** `pyproject.toml`
**Action:** Create

Write a PEP 621 `pyproject.toml` with:
- `[project]`: `name = "carestechs-agent-orchestrator"`, `version = "0.1.0"`, `requires-python = ">=3.12"`, description, authors.
- `[project.scripts]`: `orchestrator = "app.cli:main"`.
- `[project.dependencies]` (pinned, exact minor versions): `fastapi`, `uvicorn[standard]`, `pydantic>=2.7`, `pydantic-settings`, `sqlalchemy[asyncio]>=2.0`, `asyncpg`, `alembic`, `httpx`, `typer`, `uuid6`. Mark `anthropic` as an optional extra (`[project.optional-dependencies] anthropic = ["anthropic"]`) — the stub provider has zero SDK imports in v1.
- `[tool.uv] dev-dependencies`: `pytest`, `pytest-asyncio`, `respx`, `freezegun`, `ruff`, `pyright`.
- `[tool.orchestrator]`: empty table with a comment — reserved for `config.py`'s pyproject layer (T-003).
- `[tool.ruff]`: `line-length = 120`, `target-version = "py312"`, `[tool.ruff.lint]` enabling `E,F,I,B,UP,RUF,SIM,PT` rule sets.
- `[tool.pyright]`: `typeCheckingMode = "strict"`, `include = ["src"]`, `reportMissingImports = true`.
- `[tool.pytest.ini_options]`: `asyncio_mode = "strict"`, `addopts = "--strict-markers"`, `markers = ["live: hits real LLM/engine; off by default"]`, `testpaths = ["tests"]`.

Follow the `python-ai-agent-service-docker-compose` profile exactly — no swaps without an ADR update.

## Edge Cases & Risks

- **`orchestrator` entry resolves before `app.cli:main` exists.** `uv sync` does not execute the entry. It will only fail when `orchestrator --help` is run. Leave a one-line `def main() -> None: ...` stub in `src/app/cli.py` as part of T-002 so AC-5 can pass.
- **Ruff rule collisions with future Pydantic patterns.** `SIM` can flag Pydantic idioms. If that happens during later tasks, prefer a per-line `# noqa: SIM117` over disabling the whole rule set — the rule set matters for the rest of the code.
- **`uuid6` is a third-party dep.** Python stdlib lacks UUIDv7 until 3.13. Flagged in FEAT-001 Risks #2 — decision here is to commit to `uuid6` so `Run.id`/`Step.id`/etc. are time-sortable (meaningful for trace ordering).
