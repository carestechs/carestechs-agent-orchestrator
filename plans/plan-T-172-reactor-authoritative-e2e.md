## Overview
One integration test that drives the full 14-signal FEAT-006 flow with the engine-present path, asserts:

1. **Aux rows land via the reactor**, not via signal adapters — provable by inspecting the outbox (rows should transiently exist + then disappear).
2. **Status cache updates come from webhooks**, not from signal adapters — provable by asserting that a signal's 202 response + immediate read shows the *prior* status, while a follow-up read after `await_reactor` sees the new status.
3. **Every transition fires at least one effector** or is `no_effector`-exempt — provable by inspecting the trace stream for `effector_call` entries on every transition.

Runs under `--run-requires-engine` (opt-in, already gated in conftest).

```python
async def test_aux_rows_flow_through_outbox(
    client: AsyncClient, api_key: str, db_session: AsyncSession,
) -> None:
    # Open a work item (brief-approved triggers task-generation effector).
    r = await client.post(
        "/api/v1/work-items",
        json={"externalRef": "FEAT-R1", "type": "FEAT", "title": "E2E"},
        headers=_h(api_key),
    )
    wi_id = r.json()["data"]["id"]
    await client.post(f"/api/v1/work-items/{wi_id}/brief-approved", ...)

    # await_reactor for tasks generated by effector
    await await_reactor(
        db_session,
        lambda s: s.scalars(select(Task).where(Task.work_item_id == wi_id)).all(),
        description="tasks generated",
    )

    # Approve first task → signal commits + outbox enqueued
    task = (await db_session.scalars(
        select(Task).where(Task.work_item_id == wi_id)
    )).first()
    await client.post(f"/api/v1/tasks/{task.id}/approve", json={}, headers=_h(api_key))

    # Within a tight window, we should either see:
    # (a) the PendingAuxWrite row, or
    # (b) the Approval row already materialized (race-free if reactor runs fast)
    # Either is correct — we just assert the final state is the Approval row.
    await await_aux_row_count(db_session, Approval, task.id, minimum=1)

    # Outbox should be drained
    orphans = await db_session.scalars(
        select(PendingAuxWrite).where(PendingAuxWrite.entity_id == task.id)
    )
    assert list(orphans) == [], "outbox still has rows after reactor ran"
```

```python
    # Drive full flow (reuse e2e helper)

    # Collect all effector_call trace entries
    entries = await trace_store.read_all(filter_kind="effector_call")

    # Expected transition set — derived from the declarations
    from app.modules.ai.lifecycle.declarations import (
        work_item_workflow, task_workflow,
    )
    from app.modules.ai.lifecycle.effectors.base import no_effector_exemptions

    expected_transitions = _transition_set(work_item_workflow, task_workflow)

    # For each transition in the declared set, assert either:
    # - a trace entry with matching transition, OR
    # - the transition is in no_effector_exemptions.
    for transition_key in expected_transitions:
        has_effector_trace = any(
            t["transition_key"] == transition_key for t in entries
        )
        is_exempt = transition_key in no_effector_exemptions
        assert has_effector_trace or is_exempt, (
            f"transition {transition_key} fired neither an effector "
            f"nor is exempt — coverage gap at runtime"
        )
```

## Edge Cases & Risks
- **Flakiness from `await_reactor` timeouts.** 5s default may be tight on slow CI. If the test flakes, increase the timeout or investigate reactor latency. Do not add sleeps — the helper is the polling mechanism.
- **Trace store reads.** The JSONL trace store needs a "read all entries" helper that filters by kind. If missing, extend (trivial — iterate lines, json.loads, filter). Otherwise the test cannot prove invariant 3.
- **Engine fixture availability.** The test only runs under `--run-requires-engine`. Your existing `tests/integration/test_feat006_rc2_real_engine.py` already uses this gate — reuse its setup if possible (engine URL, tenant key, workflow bootstrap expectation).
- **Test length.** This will be the longest single test in the suite (14 signals + assertions). Keep assertions precise and move shared rigging into `_reactor_helpers.py` or a new `_feat008_helpers.py`. Don't let it become a 500-line wall.
- **Invariant 3's transition_key comparison.** Requires that effector trace entries include `transition_key`. Confirm T-161's trace payload shape includes it; if not, T-171/T-161 need to add it. Do not skip this — without it, invariant 3 is unenforceable from a test.
